# 后续开发待办

这份清单用于下一轮继续开发时快速接上。

## 高优先级

- 检查 README 编码，如果仍然乱码，重写为干净 UTF-8 中文 README。
- 用浏览器或实机再确认顶部固定栏高度在首页、我的页面、搜索结果、歌单详情、歌手详情、专辑详情中是否一致。
- 继续观察喜欢歌曲状态：重启、切歌、歌单刷新、搜索结果播放、播放详情页切换时都要一致。
- Android 实机验证最新 APK：启动速度、网易云 API 子服务、播放、登录、返回键、播放队列、歌词。

## 中优先级

- 把顶部控制区相关 CSS 继续收敛成更明确的公共规则。
- 把歌曲行组件的按下态、序号、歌曲数量、右侧操作按钮对齐再做一次统一审查。
- 给专辑详情页补更多测试：从歌手详情进入、列表模式渲染、歌曲播放、返回。
- 给播放详情歌词交互补测试：未选中时点击播放按钮区域不快进，选中后才快进。

## 低优先级

- 整理 AI/TTS 设置页文案和 fallback 状态提示。
- 增加部署文档里的 Dockerfile 或 systemd 示例。
- 增加移动端截图回归检查流程。

## 接手检查清单

每次重新接手项目，先做：

```powershell
git status --short --branch
npm.cmd run check:css
```

如果要改 JS 或打包：

```powershell
npm.cmd run check
```

如果要打 APK：

```powershell
$env:npm_config_disturl='https://nodejs.org/download/release'
npm.cmd run android:build:debug
```

## 给后续协作模型的提示

- 不要恢复旧乱码文档，当前这套文档就是新的基准。
- 用户会持续根据截图微调 UI，先看图再改代码。
- 不要把一个页面的问题写成单页特殊规则，能全局统一就全局统一。
- 每次改动后说清楚验证命令和结果，不要只说“应该好了”。
